prototyping in ruby was a great way to prove the concept but my main motivation for this project was to play some more with pig.

the main relation we want to model is the idea of a trend ( one record per facet ).

are p's required when we paste in wordpress?! to calculate whether a new value is trending or not we need to be able to calculate the mean and standard deviation of the time series and there are two ways to do this; the first is the naive way our ruby implementation has done it. keep track of all frequencies seen so far and recalculating as required. this is simple but is not constant in space (alarm bells!) a more elegant way is to keep track of a few key components and fold new values in. the mean at time n+1 can be calculated from the mean at time n (if you keep also keep track of n) eg mean at time n+1 = ( (mean at time n * n) + new value ) / n+1 the standard deviation at time n+1 can be similiarly calculated if you keep track of the mean of the square of the values. so our main relation has the schema 
{key: chararray,m: double,ms: double,n: int}
